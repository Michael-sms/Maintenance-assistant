# 知识图谱课设项目计划（两人小组）

## 1. 项目目标与应用场景
- **项目目标**：基于“知识图谱构建数据集”构建航空器领域知识图谱，并实现一个可用的知识图谱应用。
- **项目应用名称**：飞行器维修助手
- **应用场景**：**飞行器维修知识问答 + 典型故障定位辅助**
  - 用户输入故障现象或部件名称，系统返回可能关联部件、故障原因、维修步骤与关联关系路径。
  - 支持实体检索、关系路径查询、故障-部件-维修方案联查。

## 2. 数据与范围
- **数据来源**：`知识图谱构建数据集/` 目录下的三类文档：
  - `变构飞行器/`
  - `飞行器/`
  - `维修类/`
- **知识规模目标**（满足双人组要求）：
  - 实体（概念）≥ 500
  - 关系 ≥ 1000

## 3. 成员分工
- **成员 A（组长）**：
  - 设计本体与Schema（实体类型、关系类型、属性字段）
  - 负责信息抽取流水线：文本清洗、分句、实体/关系抽取、初步去重
  - 负责图谱入库与查询接口（图数据库或三元组存储）
- **成员 B**：
  - 负责应用层与展示：知识问答/故障联查界面、查询结果可视化
  - 负责标注样例与评估集整理（至少 200 个概念，400+ 关系）
  - 负责技术报告与演示视频素材整理

## 4. 具体实现内容
### 4.1 知识图谱构建
1. **数据预处理**：
   - 文本清洗（去噪、编码统一、段落切分）
   - 术语词表整理（部件、故障、系统、维修动作等）
2. **实体抽取**：
   - 规则+词典+模型混合方式（避免仅使用 LLM）
   - 实体类型：飞行器、部件、系统、故障现象、故障原因、维修动作、材料/工艺
3. **关系抽取**：
   - 规则模板（如“导致”“由…组成”“适用于”）
   - 依存/句法特征 + 词典触发
4. **融合与去重**：
   - 同义词合并、别名维护
5. **入库与查询**：
   - 图数据库或三元组存储
   - 支持实体检索与关系路径查询

### 4.2 知识图谱应用
- **应用名称**：飞行器维修助手
- **核心功能**：
  1. **实体检索**：输入实体名，返回类型、属性与邻接关系
  2. **故障联查**：输入故障现象，返回可能原因、关联部件与维修动作
  3. **路径查询**：显示“故障 → 部件 → 维修方案”路径
- **展示形式**：Web 界面或轻量 UI + 图谱可视化

### 4.3 评估与报告
- **评估集**：人工标注至少 200 概念、400+ 关系
- **指标**：准确率/召回率/F1（针对抽取）
- **文档与视频**：技术报告 + 4-5 分钟讲解视频

## 5. 计划里程碑
- **第 1 周（5/12-5/15）**：完成本体设计、数据清洗、首批抽取与入库
- **第 2 周**：完善抽取规则/模型、扩大规模、完成应用雏形
- **第 3 周**：评估、报告整理、视频录制
